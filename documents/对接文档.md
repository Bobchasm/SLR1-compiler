# 语义树中间代码生成指南

## 概述

本文档说明如何从新的语义树结构生成LLVM IR中间代码。语义树使用`semanticChildren`和`semanticType`来组织节点，与原来的完整语法树（`children`）不同。

---

## 核心变化

### 1. 遍历方式改变

**旧方式（遍历完整语法树）：**
```cpp
for (auto child : node->children) {
    visit(child);
}
```

**新方式（遍历语义树）：**
```cpp
for (auto child : node->semanticChildren) {
    visitNode(child);
}
```

### 2. 节点类型判断

**旧方式（基于symbol字符串）：**
```cpp
if (node->symbol == "varDecl") {
    // 处理变量声明
}
```

**新方式（基于semanticType）：**
```cpp
if (node->semanticType == "VarDecl") {
    // 处理变量声明
}
```

---

## 语义节点类型及处理方法

### 1. Program（程序根节点）

**结构：**
```
Program
├── VarDecl (全局变量)
├── VarDecl (全局变量)
├── FunctionDef (函数定义)
└── FunctionDef (函数定义)
```

**处理方式：**
```cpp
void visitProgram(ParseTreeNode* node) {
    // 遍历所有全局声明和函数定义
    for (auto child : node->semanticChildren) {
        if (child->semanticType == "VarDecl") {
            visitGlobalVarDecl(child);
        } else if (child->semanticType == "FunctionDef") {
            visitFunctionDef(child);
        }
    }
}
```

---

### 2. VarDecl（变量声明）

**节点属性：**
- `varType`: 变量类型（"int", "float", "void"）
- `varName`: 变量名
- `isGlobal`: 是否全局变量
- `isConst`: 是否常量
- `initValue`: 初始化值的字符串表示（简单值）
- `semanticChildren`: 初始化表达式的语义树（复杂表达式）

**结构示例：**
```
VarDecl: local int a = 1
└── Number: 1

VarDecl: local int g
└── BinaryExpr: +
    ├── Variable: b
    └── BinaryExpr: *
        ├── Variable: a
        └── Variable: c

VarDecl: local int c = 32
└── UnaryExpr: -
    └── Number: 32
```

**处理方式：**
```cpp
void visitVarDecl(ParseTreeNode* node) {
    // 获取变量信息
    string varType = node->varType;      // "int", "float"
    string varName = node->varName;
    bool isGlobal = node->isGlobal;
    bool isConst = node->isConst;
    
    // 转换类型
    Type* llvmType = getLLVMType(varType);
    
    // 计算初始化值
    Value* initValue = nullptr;
    if (!node->semanticChildren.empty()) {
        // 有复杂初始化表达式，遍历表达式树
        initValue = visitExpression(node->semanticChildren[0]);
    } else if (!node->initValue.empty()) {
        // 简单常量值
        initValue = ConstantInt::get(node->initValue, context);
    }
    
    // 生成LLVM IR
    if (isGlobal) {
        // 全局变量
        GlobalVariable* gv = GlobalVariable::create(
            varName, module, llvmType, isConst, initValue
        );
        symbolTable.put(varName, gv);
    } else {
        // 局部变量
        AllocaInst* alloca = builder->createAlloca(llvmType, varName);
        symbolTable.put(varName, alloca);
        
        if (initValue) {
            builder->createStore(initValue, alloca);
        }
    }
}
```

### 3. FunctionDef（函数定义）

**节点属性：**
- `varType`: 返回类型（"int", "float", "void"）
- `varName`: 函数名
- `semanticChildren`: 包含参数（FuncParam）和函数体语句

**结构示例：**
```
FunctionDef: int add()
├── FuncParam: int num1
├── FuncParam: int num2
├── FuncParam: int mod
└── ReturnStmt: BinaryExpr
    └── BinaryExpr: %
        ├── BinaryExpr: +
        │   ├── Variable: num1
        │   └── Variable: num2
        └── Variable: mod
```

**处理方式：**
```cpp
void visitFunctionDef(ParseTreeNode* node) {
    string returnType = node->varType;
    string funcName = node->varName;
    
    // 分离参数和函数体
    vector<ParseTreeNode*> params;
    vector<ParseTreeNode*> body;
    
    for (auto child : node->semanticChildren) {
        if (child->semanticType == "FuncParam") {
            params.push_back(child);
        } else {
            body.push_back(child);
        }
    }
    
    // 创建函数参数类型列表
    vector<Type*> paramTypes;
    vector<string> paramNames;
    for (auto param : params) {
        paramTypes.push_back(getLLVMType(param->varType));
        paramNames.push_back(param->varName);
    }
    
    // 创建函数
    Type* retType = getLLVMType(returnType);
    FunctionType* funcType = FunctionType::get(retType, paramTypes);
    Function* func = Function::create(funcType, funcName, module);
    
    // 创建入口基本块
    BasicBlock* entryBB = BasicBlock::create("entry", func);
    builder->setInsertPoint(entryBB);
    
    // 处理参数：为每个参数创建alloca并存储参数值
    auto argIt = func->arg_begin();
    for (size_t i = 0; i < params.size(); i++, argIt++) {
        AllocaInst* alloca = builder->createAlloca(
            paramTypes[i], paramNames[i]
        );
        builder->createStore(argIt, alloca);
        symbolTable.put(paramNames[i], alloca);
    }
    
    // 处理函数体
    for (auto stmt : body) {
        visitNode(stmt);
    }
    
    // 如果没有返回语句，添加默认返回
    if (!builder->getCurrentBlock()->hasTerminator()) {
        if (returnType == "void") {
            builder->createRetVoid();
        } else {
            builder->createRet(getDefaultValue(retType));
        }
    }
}
```

### 5. Assignment（赋值语句）

**节点属性：**
- `varName`: 被赋值的变量名
- `semanticChildren[0]`: 右侧表达式

**结构示例：**
```
Assignment: a = Number
└── Number: 32

Assignment: b = BinaryExpr
└── BinaryExpr: +
    ├── Variable: a
    └── Number: 1
```

**处理方式：**
```cpp
void visitAssignment(ParseTreeNode* node) {
    string varName = node->varName;
    
    // 计算右侧表达式
    Value* rhs = visitExpression(node->semanticChildren[0]);
    
    // 获取变量地址
    Value* varPtr = symbolTable.get(varName);
    
    // 生成store指令
    builder->createStore(rhs, varPtr);
}
```

### 6. IfStmt（if语句）

**重要：结构化的条件语句**

**节点属性：**
- `semanticChildren[0]`: Condition节点
- `semanticChildren[1]`: ThenBranch节点
- `semanticChildren[2]`: ElseBranch节点（可选）

**结构示例：**
```
IfStmt: 
├── Condition: 
│   └── UnaryExpr: !
│       └── Number: 0
├── ThenBranch: 
│   └── Assignment: a = Number
│       └── Number: 2
└── ElseBranch: 
    └── Assignment: b = Number
        └── Number: 3
```

**else if 结构示例：**
```
IfStmt: 
├── Condition: 
│   └── BinaryExpr: >
├── ThenBranch: 
│   └── ...
└── ElseBranch: 
    └── IfStmt:           <- 嵌套的if语句
        ├── Condition: 
        ├── ThenBranch: 
        └── ElseBranch:
```

**处理方式：**
```cpp
void visitIfStmt(ParseTreeNode* node) {
    // 获取三个部分
    ParseTreeNode* conditionNode = node->semanticChildren[0];
    ParseTreeNode* thenNode = node->semanticChildren[1];
    ParseTreeNode* elseNode = node->semanticChildren.size() > 2 
                               ? node->semanticChildren[2] 
                               : nullptr;
    
    // 计算条件表达式（从Condition节点的子节点获取）
    Value* condValue = visitExpression(conditionNode->semanticChildren[0]);
    
    // 创建基本块
    Function* currentFunc = builder->getCurrentFunction();
    BasicBlock* thenBB = BasicBlock::create("then", currentFunc);
    BasicBlock* elseBB = elseNode 
                         ? BasicBlock::create("else", currentFunc) 
                         : nullptr;
    BasicBlock* mergeBB = BasicBlock::create("merge", currentFunc);
    
    // 生成条件跳转
    builder->createCondBr(condValue, thenBB, elseBB ? elseBB : mergeBB);
    
    // 生成then分支
    builder->setInsertPoint(thenBB);
    for (auto stmt : thenNode->semanticChildren) {
        visitNode(stmt);
    }
    if (!builder->getCurrentBlock()->hasTerminator()) {
        builder->createBr(mergeBB);
    }
    
    // 生成else分支（如果有）
    if (elseNode) {
        builder->setInsertPoint(elseBB);
        for (auto stmt : elseNode->semanticChildren) {
            visitNode(stmt);
        }
        if (!builder->getCurrentBlock()->hasTerminator()) {
            builder->createBr(mergeBB);
        }
    }
    
    // 继续后续代码
    builder->setInsertPoint(mergeBB);
}
```

### 7. ReturnStmt（return语句）

**节点属性：**
- `semanticChildren`: 返回值表达式（可能为空）

**结构示例：**
```
ReturnStmt: Number
└── Number: 0

ReturnStmt: BinaryExpr
└── BinaryExpr: +
    ├── Variable: a
    └── Variable: b

ReturnStmt:    <- void函数的return
```

**处理方式：**
```cpp
void visitReturnStmt(ParseTreeNode* node) {
    if (node->semanticChildren.empty()) {
        // void返回
        builder->createRetVoid();
    } else {
        // 计算返回值表达式
        Value* retValue = visitExpression(node->semanticChildren[0]);
        builder->createRet(retValue);
    }
}
```

### 8. 表达式节点

#### 8.1 BinaryExpr（二元表达式）

**节点属性：**
- `operatorType`: 运算符（"+", "-", "*", "/", "%", "==", "!=", "<", ">", "<=", ">=", "&&", "||"）
- `semanticChildren[0]`: 左操作数
- `semanticChildren[1]`: 右操作数

**结构示例：**
```
BinaryExpr: +
├── Variable: a
└── Number: 1

BinaryExpr: *
├── BinaryExpr: +
│   ├── Variable: a
│   └── Variable: c
└── Variable: a
```

**处理方式：**
```cpp
Value* visitBinaryExpr(ParseTreeNode* node) {
    // 递归计算左右操作数
    Value* lhs = visitExpression(node->semanticChildren[0]);
    Value* rhs = visitExpression(node->semanticChildren[1]);
    
    string op = node->operatorType;
    
    // 算术运算
    if (op == "+") return builder->createAdd(lhs, rhs);
    if (op == "-") return builder->createSub(lhs, rhs);
    if (op == "*") return builder->createMul(lhs, rhs);
    if (op == "/") return builder->createSDiv(lhs, rhs);
    if (op == "%") return builder->createSRem(lhs, rhs);
    
    // 比较运算
    if (op == "==") return builder->createICmpEQ(lhs, rhs);
    if (op == "!=") return builder->createICmpNE(lhs, rhs);
    if (op == "<")  return builder->createICmpSLT(lhs, rhs);
    if (op == ">")  return builder->createICmpSGT(lhs, rhs);
    if (op == "<=") return builder->createICmpSLE(lhs, rhs);
    if (op == ">=") return builder->createICmpSGE(lhs, rhs);
    
    // 逻辑运算
    if (op == "&&") return builder->createAnd(lhs, rhs);
    if (op == "||") return builder->createOr(lhs, rhs);
    
    return nullptr;
}
```

#### 8.2 UnaryExpr（一元表达式）

**节点属性：**
- `operatorType`: 运算符（"+", "-", "!"）
- `semanticChildren[0]`: 操作数

**结构示例：**
```
UnaryExpr: -
└── Number: 32

UnaryExpr: !
└── Variable: flag

UnaryExpr: -
└── Variable: a
```

**处理方式：**
```cpp
Value* visitUnaryExpr(ParseTreeNode* node) {
    // 递归计算操作数
    Value* operand = visitExpression(node->semanticChildren[0]);
    
    string op = node->operatorType;
    
    if (op == "-") {
        // 负号：0 - operand
        Value* zero = ConstantInt::get(0, operand->getType());
        return builder->createSub(zero, operand);
    }
    if (op == "!") {
        // 逻辑非
        return builder->createNot(operand);
    }
    if (op == "+") {
        // 正号：直接返回操作数
        return operand;
    }
    
    return nullptr;
}
```

#### 8.3 FunctionCall（函数调用）

**节点属性：**
- `varName`: 函数名
- `semanticChildren`: 参数列表（每个参数是一个表达式）

**结构示例：**
```
FunctionCall: add()
├── Variable: b
└── Variable: c

FunctionCall: test()    <- 无参数
```

**处理方式：**
```cpp
Value* visitFunctionCall(ParseTreeNode* node) {
    string funcName = node->varName;
    
    // 计算所有参数
    vector<Value*> args;
    for (auto arg : node->semanticChildren) {
        args.push_back(visitExpression(arg));
    }
    
    // 获取函数
    Function* func = module->getFunction(funcName);
    
    // 生成call指令
    return builder->createCall(func, args);
}
```

#### 8.4 Variable（变量引用）

**节点属性：**
- `varName`: 变量名
- `semanticChildren`: 空（叶子节点）

**处理方式：**
```cpp
Value* visitVariable(ParseTreeNode* node) {
    string varName = node->varName;
    
    // 从符号表获取变量地址
    Value* varPtr = symbolTable.get(varName);
    
    // 生成load指令读取变量值

}
```

#### 8.5 Number（数字常量）

**节点属性：**
- `value`: 数字的字符串表示
- `semanticChildren`: 空（叶子节点）

**处理方式：**
```cpp
Value* visitNumber(ParseTreeNode* node) {
    string numStr = node->value;
    
    // 判断是整数还是浮点数
    if (numStr.find('.') != string::npos) {
        // 浮点数
        double val = stod(numStr);
        return ConstantFP::get(val, context);
    } else {
        // 整数
        int val = stoi(numStr);
        return ConstantInt::get(val, context);
    }
}
```

### 9. Condition、ThenBranch、ElseBranch（if语句的结构节点）

这些是辅助节点，**不需要单独的visitor**。它们的作用是组织IfStmt的结构：

- `Condition`: 包含条件表达式
- `ThenBranch`: 包含then分支的语句
- `ElseBranch`: 包含else分支的语句

在`visitIfStmt`中通过索引访问即可。

## 完整的visitor分发函数

```cpp
void visitNode(ParseTreeNode* node) {
    if (!node || node->semanticType.empty()) {
        return;
    }
    
    if (node->semanticType == "Program") {
        visitProgram(node);
    }
    else if (node->semanticType == "VarDecl") {
        visitVarDecl(node);
    }
    else if (node->semanticType == "FunctionDef") {
        visitFunctionDef(node);
    }
    else if (node->semanticType == "Assignment") {
        visitAssignment(node);
    }
    else if (node->semanticType == "IfStmt") {
        visitIfStmt(node);
    }
    else if (node->semanticType == "ReturnStmt") {
        visitReturnStmt(node);
    }
    // 表达式节点通过visitExpression处理
}

Value* visitExpression(ParseTreeNode* node) {
    if (!node || node->semanticType.empty()) {
        return nullptr;
    }
    
    if (node->semanticType == "Number") {
        return visitNumber(node);
    }
    else if (node->semanticType == "Variable") {
        return visitVariable(node);
    }
    else if (node->semanticType == "BinaryExpr") {
        return visitBinaryExpr(node);
    }
    else if (node->semanticType == "UnaryExpr") {
        return visitUnaryExpr(node);
    }
    else if (node->semanticType == "FunctionCall") {
        return visitFunctionCall(node);
    }
    
    return nullptr;
}
```

## 关键注意事项

### 1. 符号表管理

需要维护变量和函数的符号表，支持作用域：

```cpp
class SymbolTable {
    vector<map<string, Value*>> scopes;
    
public:
    void enterScope() {
        scopes.push_back(map<string, Value*>());
    }
    
    void exitScope() {
        scopes.pop_back();
    }
    
    void put(string name, Value* value) {
        scopes.back()[name] = value;
    }
    
    Value* get(string name) {
        // 从内层到外层查找
        for (int i = scopes.size() - 1; i >= 0; i--) {
            if (scopes[i].count(name)) {
                return scopes[i][name];
            }
        }
        return nullptr;
    }
};
```

### 2. 类型转换

需要实现LLVM类型转换辅助函数：

```cpp
Type* getLLVMType(string typeName) {
    if (typeName == "int") return Type::getInt32Ty(context);
    if (typeName == "float") return Type::getFloatTy(context);
    if (typeName == "void") return Type::getVoidTy(context);
    return nullptr;
}
```

### 3. 基本块终结符检查

在生成跳转指令前，检查当前基本块是否已有终结符：

```cpp
if (!builder->getCurrentBlock()->hasTerminator()) {
    builder->createBr(nextBlock);
}
```

### 4. 函数体作用域

在处理函数体时，需要进入新的作用域：

```cpp
void visitFunctionDef(ParseTreeNode* node) {
    // ...
    symbolTable.enterScope();  // 进入函数作用域
    
    // 处理参数和函数体
    // ...
    
    symbolTable.exitScope();   // 退出函数作用域
}
```

## 总结

新的语义树结构的优势：

1. **结构清晰**：语义信息明确，不包含语法细节
2. **易于遍历**：只需遍历`semanticChildren`
3. **类型安全**：通过`semanticType`判断节点类型
4. **便于维护**：中间代码生成逻辑简洁明了

主要变化：
- 使用`semanticChildren`代替`children`
- 使用`semanticType`代替`symbol`字符串匹配
- IfStmt有明确的Condition/ThenBranch/ElseBranch结构
- 表达式节点直接提供`operatorType`等属性
- VarDecl包含完整的初始化表达式树
