# 用来记录一些七七八八的东西

## 遇到的一些问题

1.生成语法分析表时，刚开始产生了两三个状态，非常不对劲

在构建goto等某个过程时currentState用了引用传递，而这个变量在该处只应被读，而不能导致其修改，以使前面的信息覆盖，导致整个状态的崩溃

2.生成的语法分析表存在许多列没有一个动作

开始认为是输出至csv时由于有终结符是','，与csv的分隔符冲突了，导致输出的csv有问题，后面增加了转义处理，还是很多空白
后来发现原因是词法分析器与语法分析器规定的 整型、浮点型、标识符 的字符串不一样
也就是 tokenTypeToTerminal 中没有正确映射，而是写成一样的了

3.最后输出的分析表感觉可以用了，但是查看log发现存在两个冲突

- 一个是if-else悬空，这是非常经典的 移进-规约 冲突问题，这个冲突优先shift就可以，不用管

- 还有一个是函数返回值定义和变量类型定义的 规约-规约 冲突，语法分析器不知道是规约成函数还是变量的类型定义 <br>
  
    而这个规约成哪一个其实也不太重要，这两个没必要分得那么清楚，小改了以下文法规则
  
    **冲突的两个规约：** <br>
    r26：对应产生式26（funcType -> int）<br>
    r12：对应产生式12（bType -> int）
  
    **改产生式：**<br>
    原来：bType -> 'int' | 'float' <br>
    改成：bType -> 'int' | 'float' | 'void' <br>
    再删除：funcType -> 'void' 和 funcType -> 'int' <br>
    再修改： funcDef，将函数定义改为使用 bType

3.就是本来test,test1这些是正确用例可以构建出语法分析树的，但是昨天突然某次开始偶尔构建不出来了，再到后面只剩test4是正常的，而且代码我没动过，还回退了几次到原来的版本测。然后单独测试lexer，token输出正常，但是加上parser，token输出最后一行莫名其妙多出来一个从来没出现的标识符neIntel \<IDN,neIntel\>导致语法分析出错，怀疑是某个地方操作char*的时候漏了\0，这种内存或者缓冲区问题，然后调调调，加强了判断文件末尾的判断还是不行。刚刚调试着看文件大小，可能是读用例的时候，把内容弄到input时就有问题，以及一些内存没清干净，然后改了一下输入文件处理，初始化的时候加了一些clean操作等缓存区操作
