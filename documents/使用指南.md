  编译器使用指南

   快速开始

    1. 编译项目

```bash
  进入项目目录
cd c:\Users\blc\Desktop\compiler

  清理之前的构建
mingw32-make clean

  编译完整编译器
mingw32-make compiler
```

编译成功后会在`build/`目录下生成：
- `lexer.exe` - 词法分析器
- `parser.exe` - 语法分析器  
- `compiler.exe` - 完整编译器

    2. 运行测试

     方法一：使用测试脚本（推荐）

```bash
  运行所有测试
run_all_tests.bat

  只运行前端测试（语法分析）
test_frontend.bat

  只运行中端测试（IR生成）
test_backend.bat
```

     方法二：手动测试单个文件

```bash
  测试语法分析（前端）
build\parser.exe case\frontend_test1_basic.sy

  测试IR生成（完整流程）
build\compiler.exe case\backend_test1_simple.sy
```

    3. 查看输出

运行后会在`case/`目录生成多个输出文件：

```
case/
├── test1.sy                        输入的源文件
├── test1_token_result.txt          词法分析结果
├── test1_parse_analysis.txt        语法分析过程
├── test1_parse_tree.md             语法树（Mermaid格式）
├── test1_semantic_tree.txt         语义树
└── test1_output.ll                 LLVM IR代码
```

   详细使用说明

    词法分析器

```bash
  编译
mingw32-make lexer

  运行
build\lexer.exe case\test.sy

  输出文件
case\test_token_result.txt
```

**输出格式示例**:
```
int    <KW,1>
main   <KW,5>
(      <SE,23>
)      <SE,24>
{      <SE,25>
...
```

    语法分析器

```bash
  编译
mingw32-make parser

  运行
build\parser.exe case\test.sy

  输出
- 终端: Accept 或 Error
- 文件: case\test_parse_analysis.txt
       case\test_parse_tree.md
       case\test_semantic_tree.txt
```

**语法分析过程示例** (`*_parse_analysis.txt`):
```
Step    State       Symbol Stack        Input               Action         
---------------------------------------------------------------------------
1       0                               int                 s11            
2       11          int                 Ident               r12            
3       2           bType               Ident               s14            
...
```

**语义树示例** (`*_semantic_tree.txt`):
```
Program
└── FunctionDef: int main()
    ├── VarDecl: local int a = 1
    │   └── Number: 1
    ├── Assignment: a = BinaryExpr
    │   └── BinaryExpr: +
    │       ├── Variable: a
    │       └── Number: 1
    └── ReturnStmt: Number
        └── Number: 0
```

    完整编译器

```bash
  编译
mingw32-make compiler

  运行
build\compiler.exe case\test.sy

  输出
- 终端: 各阶段的进度信息
- 文件: 所有分析阶段的输出 + IR文件
- 日志: logs/log_YYYYMMDD_HHMMSS.txt
```

**LLVM IR示例** (`*_output.ll`):
```llvm
define i32 @main() {
label_main_ENTRY:
  %a = alloca i32
  store i32 1, i32* %a
  %op0 = load i32, i32* %a
  %op1 = add i32 %op0, 1
  store i32 %op1, i32* %a
  ret i32 0
}
```

   调试技巧

    1. 查看详细日志

每次运行`compiler.exe`或`parser.exe`都会在`logs/`目录生成日志文件：

日志包含：

- DFA构建过程
- 每一步的分析决策
- 错误详情

    2. 查看中间输出

```bash

type case\test_token_result.txt

type case\test_parse_analysis.txt

  查看语义树
type case\test_semantic_tree.txt

  查看生成的IR代码
type case\test_output.ll
```

    3. 可视化语法树

`*_parse_tree.md`文件包含Mermaid格式的语法树，可以：

1. 用支持Mermaid的Markdown编辑器打开（如Typora）
2. 复制到在线Mermaid编辑器：https://mermaid.live/
3. 使用VSCode的Mermaid插件查看

    4. 常见问题

**Q: 编译失败，提示找不到g++**

A: 确保MinGW已安装并添加到PATH环境变量：
```bash
g++ --version    检查是否能找到g++
```

**Q: 语法分析失败**

A: 检查`logs/`目录下最新的日志文件，查看详细错误信息

**Q: IR生成失败或输出为空**

A: 确保测试用例没有使用不支持的特性：
- ❌ 避免使用float类型
- ❌ 避免使用局部const
- ❌ 避免使用一元运算符（+a, -a, !a）

   创建自己的测试用例

    1. 前端测试（验证语法）

创建文件`case\my_test.sy`：

```c
// 可以使用所有特性
int main() {
    float x = 3.14;  // 浮点数 ✓
    const int y = 10; // 局部常量 ✓
    int z = -y;      // 一元运算符 ✓
    return 0;
}
```

测试：
```bash
build\parser.exe case\my_test.sy
```

期望输出：`Accept`

    2. 中端测试（生成IR）

创建文件`case\my_ir_test.sy`：

```c
// 只使用支持的特性
int main() {
    int a = 10;      // 整数 ✓
    int b = 20;
    int c = a + b;   // 算术运算 ✓
    
    if (c > 25) {    // 控制流 ✓
        return 1;
    }
    return 0;
}
```

测试：
```bash
build\compiler.exe case\my_ir_test.sy
```

查看生成的IR：
```bash
type case\my_ir_test_output.ll
```
